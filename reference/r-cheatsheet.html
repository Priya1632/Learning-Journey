R Coding Reference

The running cheat sheet I wish I’d had on day one

A living reference. Things go here when I learn them and want to find them fast later. Not exhaustive, opinionated about what’s useful for biostat / HEOR work.

The non-negotiables

  • R is case-sensitive. Always. Filter ≠ filter. df ≠ DF.
  • Assignment uses <- by convention, though = works in most places. Inside function calls, = means “named argument,” not “assign.”
  • == is equality, = is assignment-in-arguments. Mixing them is a classic SAS-programmer bug.
  • Vectors are 1-indexed, not 0. x[1] is the first element.
  • NA is contagious. Most arithmetic with NA returns NA. Use na.rm = TRUE in summary functions, or sum(is.na(x)) to count missings.

Pipes

  • |> is the base R pipe (R 4.1+). Prefer it for new code.
  • %>% is the magrittr pipe. Almost identical. Some edge cases differ — %>% supports . placeholder, |> supports _ placeholder (with named args only) starting R 4.2.

Functions vs. macros (the SAS programmer’s mental shift)

  • Functions in R are first-class objects. You can pass them as arguments, store them in lists, return them from other functions. SAS macros are text substitution; R functions are real values.
  • Default arguments in functions: f <- function(x, n = 10) .... The n = 10 is the default if not supplied.
  • The ... (dots) argument captures additional arguments to pass to inner functions. Used heavily in tidyverse and for plotting.

Common gotchas

  • Function name conflicts. Loading dplyr masks stats::filter() and stats::lag(). Loading MASS masks dplyr::select(). When in doubt, prefix: dplyr::filter(). The conflicted package makes this explicit and is worth using.
  • data.frame vs tibble vs data.table. All three are tabular. Tibbles print nicer and don’t do silent type coercion. data.table is fast but a different syntax. For tidyverse work, tibbles. For huge data, data.table or arrow.
  • Factors. read.csv used to make strings into factors silently. Modern R (4.0+) doesn’t. readr::read_csv and data.table::fread never did. Factors are still useful — just don’t get them by accident.
  • sapply is a foot-gun. It tries to simplify the output and can return a vector, matrix, or list depending on inputs. Use vapply (type-safe) or purrr::map_* (explicit return type) instead.

Useful idioms

# Read everything as character to avoid type surprises, then convert deliberately
readr::read_csv("file.csv", col_types = readr::cols(.default = "c"))

# "Fill down" missing values (very common in pivoted data)
tidyr::fill(df, col_name, .direction = "down")

# Per-group operation without group_by/ungroup ceremony (dplyr 1.1+)
df |> mutate(rank = row_number(), .by = subject_id)

# Anti-join: rows in x not in y by key
dplyr::anti_join(x, y, by = "id")

# String interpolation
glue::glue("Subject {usubjid} had Cmax = {round(cmax, 2)}")

Project setup

  • Always use an RStudio Project (.Rproj file). Sets the working directory automatically — no more setwd().
  • here::here() for paths. Resolves from the project root, not from wherever the script happens to be running.
  • renv for project-level package management. Like a requirements.txt for R. Essential for reproducible analyses.
  • targets for analysis pipelines. The make-equivalent for R; reruns only what changed. Worth learning for any multi-step project.

Table 1 / regulatory output

  • gtsummary::tbl_summary() — the fastest path to a publication-ready Table 1.
  • gt — the underlying table package; full control.
  • tfrmt (pharmaverse) — for regulatory-compliant tables matching SAS-style specs.

Modeling

  • broom::tidy() turns any model object into a tidy data frame of coefficients. broom::glance() for model-level stats. broom::augment() for fitted/residual values.
  • marginaleffects is the right way to get adjusted predictions, marginal effects, and contrasts from any model. Replaces emmeans for most use cases I care about.
  • Always check confint() — base R confidence intervals on coefficients are profile likelihood by default for GLMs, Wald for lm. Specify if you care.

Things I keep relearning

I’ll add to this list as I keep tripping on things.

  • The default contrasts for unordered factors are treatment contrasts (dummy coding). For ordered factors, polynomial. Set options(contrasts = c("contr.treatment", "contr.treatment")) if you want consistent behavior.
  • seq(1, 10, by = 2) not seq(1, 10, 2) — positional args work but named is clearer.
  • paste() defaults to space separator; paste0() defaults to no separator. Use paste0 for filenames and identifiers.
